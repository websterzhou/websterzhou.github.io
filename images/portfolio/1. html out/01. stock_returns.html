1. Introduction

In financial economics, a key area of interest is to understand asset pricing and to satisfactorily model the behaviour of assets based on a given information set. Although traditional econometric techniques such OLS regressions are greatly used in establishing causal relationships between variables, their forecasting capabilities are notoriously weak.

With the rise of big data machine learning, there is a strong push to leverage these methods in financial economics. While a wide range of information is available for listed companies, it is only in recent years that analysts are studying these data with such deep scrutiny. On that note, this essay will use machine learning methods to forecast stock returns.

2. Data

In order to produce forecasts, various financial indicators are pulled from 10-K fillings of publicly traded companies in US over the course of 6 years from 2014-2019. The data was directly downloaded from here.

2.1. Cleaning data

To better understand the upcoming analyses, it will be useful to understand some of the variables that will be used:

  1. X1 is a vector that contains the company names of every stock.
  2. Class (later renamed to invest) is a dummy variable that is 1 if a given stock sees positive growth in period \((t+1)\) and 0 otherwise. It can be interpreted as an indicator for whether the stock is worth investing in hindsight.
  3. lagged_growth is a lagging variable that displays the percentage price variation of a given stock in the previous year.

The variable invest will be used as the outcome variable; different models will be trained to predict the value of invest for a stock based on the given set of financial indicators.

3. Analysis

5 models will be used to forecast the outcome variable invest:

  1. Lasso
  2. Adaptive lasso
  3. Elastic Net
  4. Adaptive Elastic Net
  5. Random Forest

The choice of methods stems from the nature of this dataset and the characteristics of each method. While OLS models are often used as a baseline model for comparing to other more complex models, this analysis will not do so because of its poor model selection capabilities (under rejecting insignificant variables); the same argument applies to the ridge regression, which has no element of model selection at all.

3.1. Lasso and adaptive lasso

Lasso regression has a \(\ell_1\) penalisation that helps in model selection, while the adaptive lasso is a natural extension of the lasso and its performance will be evaluated too. Here, the optimal penalisation parameter \(\lambda^*\) is chosen using the cross-validation method.

# Lasso
cv_lasso= cv.glmnet(y = y_train$invest,
                    x = x_train,
                    type.measure = "class",
                    family="binomial"
                    )


# Adaptive Lasso
omega= 1/ abs(coef(cv_lasso,s = "lambda.min"))
omega[omega==Inf] = 9999999


cv_adalasso = cv.glmnet(y = y_train$invest,
                     x = x_train, 
                     penalty.factor=omega,
                     type.measure = "class",
                     family="binomial")

3.2. Elastic net and adaptive elastic net

The elastic net methods combine both \(\ell_1\) and \(\ell_2\) penalities and may outperform \(\ell_1\) penalisation methods in some cases. Therefore, it is worth including them as well. \(\alpha\) is set at \(0.5\) to allow 50% weightage on both \(\ell_1\) and \(\ell_2\) penalisations.

#Elastic Net
enet= cv.glmnet(y = y_train$invest,
                    x = x_train,
                    type.measure = "class",
                    family="binomial",
                    alpha= 0.5
                    )


#Adaptive Elastic Net
omega2= 1/ abs(coef(enet,s = "lambda.min"))
omega2[omega2==Inf] = 9999999


aenet= cv.glmnet(y = y_train$invest,
                    x = x_train,
                    type.measure = "class",
                    family="binomial",
                    alpha= 0.5,
                    penalty.factor=omega2
                    )

3.3. Random forest

One key drawback with penalised estimation methods is the imposition of a linear structure of the outcome variable: ie. \(y_i = \boldsymbol{X}_i \boldsymbol{\beta} + \epsilon_i\). That is a strong assumption that may not hold in practice. The random forest is a model that incorporates non-linearities, ie. \(y_i = f(x_{i1} +...+x_{ik}) + \epsilon_i\); hence it is chosen as a supplement to other methods.

#Random Forest

Beyond these metrics, one can also investigate the recall-precision curve and ROC using predicted probabilities. As the number of positive observations stands in the majority, recall-precision is less useful because the ability to correctly detect negative samples is more important than correctly detecting positive ones. Hence, the ROC (depicted using the ROC curve), which plots sensitivity (true positive rate) against (\(1-\)specificity) (false positive rate) for a given threshold \(\theta\), is better suited for this dataset.

The ROC curves are plotted in Figure 2. A better model is indicated by a larger AUC (area under curve). It is apparent that the adaptive versions of lasso and elastic net perform poorly as compared to non-adaptive and random forest models. This could be because of the shrinkage effect caused by the weights, shrinking variables that are small in magnitude (but still significant) to 0 in the adaptive models. The performance of random forest, lasso and elastic net appears to be similar, consistent with findings from Table 1.

5. Conclusion

Overall, the lasso, elastic net models perform better than the adaptive versions, and the random forest appears to have the edge in overall performance. Model selection ultimately lies in the users objectives and appetite for risk. If one cares less about losses and wants a model that is best in picking out good stocks, the lasso and elastic net models are optimal.
